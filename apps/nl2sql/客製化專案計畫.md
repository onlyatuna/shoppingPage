# 客製化自然語言轉 SQL 的 Python 入口網站 - 專案計畫

根據您的需求，我們對原始範例計畫進行了調整，以下是專屬於您的架構與設計計畫：

---

## 1. 架構設計

```text
用戶輸入自然語言
       ↓
Python Web App (FastAPI) 
       ↓
Gemini API (NL → SQL 轉換) + 資料庫 Schema 注入 (RAG)
       ↓
SQL Server 執行 (使用 SQL Server 驗證)
       ↓
結果呈現 (動態表格 / 圖表 / Excel 匯出)
```

## 2. 技術選型

| 元件 | 決定技術 | 選擇理由 |
| --- | --- | --- |
| **Web 框架** | **FastAPI** | 輕量、高效能、支援異步處理 (Async)，適合串接 API。 |
| **AI 模型** | **Gemini API** | 採用 Google 的 Gemini 模型進行自然語言轉 SQL 的推理。 |
| **DB 驅動** | **pyodbc / SQLAlchemy** | 穩定支援 Microsoft SQL Server 查詢與連線。 |
| **前端技術** | **HTMX + Bootstrap** | 輕量化，不需複雜的前後端分離架構即可實現動態互動（例如：載入中提示、動態表格）。 |
| **Schema 快取** | **JSON / SQLite** | 啟動時預先載入資料表結構，以利快速注入給 Gemini 的 Prompt。 |

---

## 3. 前端頁面佈局 (初期規劃)

頁面將維持簡潔實用的五大區塊（後續可依需求彈性微調）：
1. **抬頭標題**：例如「Gemio ERP 智慧查詢系統」。
2. **對話/輸入框**：讓使用者輸入自然語言查詢需求。
3. **查詢結果區**：
   - 使用 Grid 呈現查詢結果。
   - 若有數字欄位會自動加總。
   - 提供「**匯出 Excel**」的按鈕。
4. **SQL 語句顯示區**：顯示 Gemini 產生的 SQL 語法，方便開發與除錯。
5. **可查詢表單區**：列出目前系統支援查詢的資料表清單（如：科目餘額表、採購明細表等）。
> **互動體驗**：按下查詢按鈕後，會透過 HTMX 顯示「系統查詢中...」的載入提示。

## 4. 核心流程與資安設計

**1. Schema 注入 (RAG 核心流程)**
- **啟動階段**：FastAPI 啟動時，會自動連線至 SQL Server 抓取最新的 Schema (資料表結構與欄位定義)。
- **查詢階段**：使用者提問時，會將對應的 Schema 與使用者的問題打包成 Prompt，發送給 Gemini API。
- **執行階段**：Gemini 回傳 SQL 後，FastAPI 將 SQL 語法送交 SQL Server 執行，並將回傳數據渲染到前端。

**2. 安全的資料庫連線 (SQL Server 驗證)**
為了安全起見，資料庫的連線資訊 **絕對不會** 寫死在程式碼中。將採用 `.env` 環境變數檔來管理：
```env
# .env 設定檔範例 (不會上傳到 GitHub)
DB_HOST=您的_SQL_Server_IP_或網址
DB_PORT=1433
DB_NAME=您的資料庫名稱
DB_USER=您的自訂帳號
DB_PASSWORD=您的密碼
GEMINI_API_KEY=您的_Gemini_API_金鑰
```

## 5. 發佈與部署環境

1. **版本控制 (GitHub)**：
   - 專案將推送到您**自己的 GitHub 儲存庫 (Repository)**。
   - 將設定 `.gitignore` 以防止敏感資料（如 `.env`）被上傳。
2. **主機部署 (GCP Server)**：
   - 將部署到您現有的 **GCP (Google Cloud Platform) Server** 上。
   - 會使用 `uvicorn` 搭配自訂的 Port（例如：`8080` 或 `8000`）來啟動 FastAPI 服務：
     `uvicorn main:app --host 0.0.0.0 --port 您的自訂Port`
   - （可選）後續可透過 Systemd 或 Docker 確保服務在 GCP 背景持續運行。
